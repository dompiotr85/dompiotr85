# Piotr Domański (DomPiotr85)

**Profil ekspercki w domenie inżynierii oprogramowania i architektury systemów informatycznych**

---

## Wprowadzenie

Niniejszy profil to moje portfolio które przedstawia bogate doświadczenie oraz multidyscyplinarne kompetencje w obszarze inżynierii oprogramowania,
architektury systemów informatycznych, analizy danych oraz technologii baz danych. Moja ekspertyza obejmuje zarówno aspekty teoretyczne, oparte na
solidnych fundamentach naukowych, jak i praktyczne zastosowanie zdobytej wiedzy w realnych projektach. Moje podejście do tworzenia oprogramowania
opiera się na metodykach inżynierii oprogramowania, z uwzględnieniem analizy wymagań, projektowania architektury, implementacji, testowania i
dokumentowania kodu.

## Domeny kompetencji i specjalizacja

### Główne obszary zainteresowań

*   **Inżynieria oprogramowania:** Specjalizacja w tworzeniu aplikacji desktopowych (frontend jak i backend), analizie danych oraz projektowaniu
efektywnych rozwiązań bazodanowych. Kompetencje obejmują zarówno paradygmat programowania strukturalnego, jak i obiektowego.

*   **Architektura systemów informatycznych:** Projektowanie skalowalnych i wydajnych systemów informatycznych z uwzględnieniem aspektów
bezpieczeństwa, niezawodności i łatwości konserwacji.

### Obszary zainteresowań badawczych

*   **Automatyzacja kompilacji:** Badania nad optymalizacją procesów budowania oprogramowania, ze szczególnym naciskiem na wykorzystanie narzędzia
CMake w celu osiągnięcia modularności, reprodukowalności i automatyzacji. Analiza porównawcza metod kompilacji i generowania kodu z uwzględnieniem
kryteriów wydajnościowych.

*   **Technologie baz danych:** Zaawansowane studia nad językiem SQL, z uwzględnieniem norm ANSI i UNICODE, oraz eksploracja systemów zarządzania
bazami danych (RDBMS). Implementacja rozwiązań bazodanowych w SQLite, Microsoft SQL Server, MySQL, MariaDB, PostgreSQL i Oracle Database,
z naciskiem na optymalizację zapytań i wydajność baz.

*   **Współbieżne programowanie obiektowe w języku Java:** Badanie zaawansowanych wzorców i bibliotek do programowania współbieżnego i wielowątkowego
w języku Java, ze szczególnym uwzględnieniem aspektów optymalizacji i unikania zakleszczeń (deadlock). Analiza wydajności i skalowalności aplikacji
wielowątkowych.

*   **Platforma macOS:** Eksploracja specyfiki zamkniętego ekosystemu macOS, z naciskiem na rozszerzanie funkcjonalności projektów Open Source, w tym
wykorzystanie platformy programistycznej Xcode oraz języka Swift do zapewnienia pełnej kompatybilności i wydajności aplikacji.

## Umiejętności techniczne

### Języki programowania

| Język programowania | Poziom znajomości | Paradygmaty |
|---|---|---|
| Pascal (oraz Free Pascal)  | Biegły  | Programowanie strukturalne  |
| Delphi  | Biegły  | Programowanie obiektowe  |
| C  | Biegły  | Programowanie strukturalne oraz systemowe  |
| C++  | Biegły  | Programowanie obiektowe, ogólnego przeznaczenia oraz współbieżne  |
| Java  | Zaawansowany  | Programowanie obiektowe, współbieżne  |
| HTML5  | Zaawansowany | Semantyczne znaczniki  |
| CSS  | Zaawansowany | Style kaskadowe  |
| SQL  | Dobre | Deklaratywny język zapytań |

*Analogia: Języki programowania to narzędzia w warsztacie inżyniera oprogramowania. Każdy język ma swoje unikalne właściwości i jest przeznaczony do
konkretnych zadań, podobnie jak różne rodzaje młotków, śrubokrętów i pił w warsztacie mechanicznym.*

### Bazy Danych

| System zarządzania bazą danych (RDBMS) | Znajomość | Aspekty praktyczne |
|---|---|---|
| SQLite | Dogłębna | Implementacja baz danych w aplikacjach, biblioteki |
| Microsoft SQL Server | Dogłębna | Optymalizacja zapytań, architektura baz danych |
| MySQL  | Dogłębna | Zarządzanie bazami danych, bezpieczeństwo |
| MariaDB  | Dogłębna | Klasteryzacja, replikacja |
| PostgreSQL  | Dogłębna | Rozszerzenia, geodane |
| Oracle Database  | Dogłębna | Zaawansowane funkcje, duże bazy danych |

*Analogia: Bazy danych to magazyny danych, a RDBMS to systemy zarządzające tymi magazynami. Dobre zarządzanie bazą danych jest kluczowe dla
efektywności aplikacji, tak jak właściwa organizacja magazynu jest kluczowa dla sprawnego funkcjonowania przedsiębiorstwa.*

### Narzędzia i technologie

*   **Systemy kontroli wersji:** Git (dogłębna znajomość), GitHub (profesjonalne wykorzystanie).
*   **Edytory i środowiska programistyczne:** Microsoft Visual Studio Code, Microsoft Visual Studio, Code::Blocks, Xcode, Embarcadero Delphi.
*   **Kompilatory:** Zbiór kompilatorów GNU (GCC, g++, gobjc, gcj, gpc), kompilator MSVC, MinGW-w64.
*   **Technologie graficzne:** GIMP, Adobe Photoshop.
*   **Konteneryzacja:** Docker (doświadczenie w tworzeniu, wdrażaniu i zarządzaniu kontenerami).
*   **Testowanie:** Pisanie testów jednostkowych i integracyjnych, z wykorzystaniem frameworków testowych (np. JUnit w Java, CUnit w C, Boost Test
Library dla C++, DUnit i DUnitX dla Delphi, FPCUnit i Tap4Pascal dla i Free Pascal).

*Analogia: Narzędzia to instrumenty, które umożliwiają szybkie i skuteczne tworzenie oprogramowania. Profesjonalny inżynier oprogramowania musi
biegle posługiwać się różnorodnymi narzędziami, podobnie jak muzyk wirtuoz sprawnie wykorzystuje swój instrument.*

## Zainteresowania rozwojowe

*   **Algorytmika:** Projektowanie algorytmów o zoptymalizowanej złożoności obliczeniowej, ze szczególnym naciskiem na rozwiązywanie problemów z
dziedziny informatyki.
*   **Technologie konteneryzacji:** Rozszerzanie wiedzy w zakresie orkiestracji kontenerów i wdrażania aplikacji w chmurze (np. Kubernetes,
Docker Swarm).
*   **Architektura systemów operacyjnych:** Zgłębianie mechanizmów działania systemów operacyjnych, w tym zarządzania zasobami, pamięcią i procesami.
*   **Silniki graficzne:** Rozwój wiedzy w zakresie tworzenia silników graficznych 2D i 3D z wykorzystaniem DirectX, OpenGL i Vulkan API, z naciskiem
na optymalizację renderowania i wykorzystanie zasobów sprzętowych.

## Metodologie i podejście

Moje podejście do projektowania oprogramowania opiera się na zasadach inżynierii oprogramowania, z uwzględnieniem iteracyjnego i inkrementalnego
procesu rozwoju. Szczególną wagę przykładam do:

*   **Analizy wymagań:** Dokładne zrozumienie potrzeb klienta i specyfikacji systemu.
*   **Projektowania architektury:** Tworzenie skalowalnych, wydajnych i łatwych w utrzymaniu rozwiązań.
*   **Implementacji:** Pisanie czystego, czytelnego i dobrze udokumentowanego kodu.
*   **Testowania:** Zapewnienie wysokiej jakości oprogramowania poprzez kompleksowe testy jednostkowe i integracyjne.
*   **Dokumentacji:** Tworzenie przejrzystej dokumentacji, ułatwiającej utrzymanie i rozwój systemu.

### Obszary współpracy 👯

Współpraca może obejmować następujące obszary, ale nie ogranicza się tylko do nich:

*   **Rozwój funkcjonalności:** Implementacja nowych funkcji i rozszerzeń w obu projektach, w oparciu o planowane roadmapy (opisane w poprzedniej
sekcji).
*   **Optymalizacja:** Analiza wydajności, identyfikacja wąskich gardeł i optymalizacja kodu w celu poprawy szybkości i efektywności.
*   **Testowanie:** Tworzenie testów jednostkowych i integracyjnych, pomagających zapewnić wysoką jakość kodu i stabilność aplikacji.
*   **Dokumentacja:** Tworzenie i aktualizacja dokumentacji technicznej, tutoriali i przykładów użycia.
*   **Projektowanie UI/UX:** Wsparcie w projektowaniu interfejsu użytkownika i poprawa doświadczenia użytkownika (szczególnie w projekcie WinRuler).
*   **Wsparcie społeczności:** Aktywny udział w dyskusjach, odpowiadanie na pytania, wspieranie nowych użytkowników i kontrybutorów.
*   **Badania i rozwój:** Eksploracja nowych technologii i metod, które mogą zostać włączone do projektów.
*   **Implementacja algorytmów:** Opracowanie i implementacja algorytmów graficznych, fizycznych i innych.
*   **Praca z bazą danych:** Optymalizacja zapytań do bazy danych i tworzenie schematu danych.
* **Wsparcie w Porcie:** Przenoszenie projektów na inne platformy (np. port WinRuler na macOS).
